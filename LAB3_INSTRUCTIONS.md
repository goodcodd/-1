# Інструкції для фіксації змін лабораторної роботи №3

## Перевірка результату

Перед фіксацією змін переконайтеся, що все працює:

1. Запустіть проєкт:
   ```bash
   npm run dev
   ```

2. Перевірте в браузері:
   - Відкрийте http://localhost:5173
   - Переконайтеся, що є навігація між сторінками
   - Перейдіть на сторінку "Методи масивів"
   - Перевірте, що відображаються:
     - ✅ Всі студенти (з стилізацією неактивних)
     - ✅ Активні студенти з балом > 60
     - ✅ Середній бал активних студентів
     - ✅ Сортування студентів (кнопки працюють)

## Фіксація змін у Git

### Крок 1: Перевірка статусу файлів

```bash
git status
```

Ви повинні побачити:
- Змінені файли: `src/App.jsx`, `src/App.module.css`
- Нові файли: `src/studentsData.js`, `src/components/pages/ArrayMethodsPage/`

### Крок 2: Додавання файлів до індексу

```bash
git add src/App.jsx
git add src/App.module.css
git add src/studentsData.js
git add src/components/pages/ArrayMethodsPage/
```

Або одним рядком:
```bash
git add src/
```

### Крок 3: Створення коміту

```bash
git commit -m "Lab 3: Add Array Methods page with map, filter, reduce, sort"
```

### Крок 4: Перевірка, що ви на гілці main

```bash
git branch
```

Якщо ви на іншій гілці, перейдіть на main:
```bash
git checkout main
```

### Крок 5: Злиття змін (якщо працювали в окремій гілці)

Якщо ви створювали окрему гілку для роботи:

```bash
# Переконайтеся, що всі зміни закомічені
git checkout main
git merge lab3-array-methods
```

### Крок 6: Відправка на GitHub

```bash
git push origin main
```

## Що має працювати

✅ **Array.map()** - відображення всіх студентів
✅ **Array.filter()** - фільтрація активних студентів з балом > 60
✅ **Array.reduce()** - обчислення середнього балу
✅ **Array.sort()** - сортування студентів за балами
✅ **Стилізація** - неактивні студенти сірі та закреслені

   - Всі студенти (10 студентів)
   - Активні студенти з балом > 60 (7 студентів)
   - Середній бал (має бути ~81.00)
   - Сортування (кнопки змінюють порядок)
